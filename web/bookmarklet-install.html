How it works

For each image, every opaque pixel is compared against all 186 legal map colours using OKLab ΔE (the same metric the Loominary editor uses). The score = % of pixels whose nearest-palette match is within ΔE ≤ 0.05 — roughly "how many pixels can the palette represent accurately?"

≥ 75% = good palette fit · 50–74% = moderate · < 50% = palette will lose a lot of colour information

Images on other origins may show "CORS – pixel data unavailable" unless the server includes Access-Control-Allow-Origin headers.

Source: bookmarklet.js in the Loominary web project. Palette data: Minecraft 1.21.4 via /loominary dumppalette.
